Generate an MVP from a natural-language brief

Submit a prompt describing what you want to build. LogicGen returns a project with pages, schema, logic graph, and connected capabilities — typically in 60–120 seconds. Stream events let your agent display progress and accept or reject the result before it's saved.

Auth Bearer token Rate 60 / hour Cost $0.15–0.30 per call Idempotent with key

Request body application/json

prompt required string A natural-language description of the app to build. Specific is better — name pages, integrations, and behaviors. Up to 4000 characters.
capabilities string[] Capabilities to include. Accepts canonical IDs (e.g. stripe, denova, gmail, anthropic). If omitted, the generator infers them from the prompt.
workspace_id required string Target workspace. The resulting project is created here. Get yours from GET /workspaces.
stream boolean When true, the response is an SSE stream with generation.step, generation.preview, and generation.done events. Defaults to false.
name string Optional project slug. If omitted, one is derived from the prompt and made unique within the workspace.
idempotency_key string UUID supplied in the Idempotency-Key header is also accepted in the body. Same key within 24h returns the original result.

Stream events SSE · 4 types

generation.step event Progress update. Fires for each major phase: parsing_prompt, drafting_schema, wiring_logic, generating_ui, connecting_capabilities, deploying. Includes pct.
generation.preview event A page or component is ready to preview. Use this to render incremental feedback in your UI before the full generation completes.
generation.done event Generation complete. Includes project_id, preview_url, and final costs. Project is unsaved until you call POST /projects/:id/save.
generation.error event Generation failed. Includes code and message. Common: prompt_too_vague, capability_unavailable, quota_exceeded.
